# 从零到能独立完成项目：我的全面工程师学习计划

这篇文章是我的长期学习计划公开版。目标不是“看完很多课”，而是成为一个基础扎实、能独立完成项目的全面工程师。

我的主项目是这个博客本身：`kpblog.cc`。后续学习 Web、后端、数据库、部署、测试、AI Agent，都尽量落到这个项目里。

## 当前定位

我目前的优势在 AI 和深度学习概念上，对 Transformer、优化器、U-Net 等有一定理解。但工程基础不够均衡，尤其是：

- 算法与数据结构薄弱。
- HTTP、数据库索引、范式等后端基础需要系统补。
- JavaScript、Promise、浏览器渲染机制需要重新打牢。
- Linux、Git、工程工具要形成日常习惯。

所以这份计划不会一上来冲 AI 项目，而是先把工程底座补起来。

## 每周节奏

我的工作是早 9 晚 5，通勤单程 1 小时多。夏天每天还要洗澡、洗衣，所以工作日晚上的深度学习不能安排得太满。

每周目标：

| 类型 | 时间 |
|---|---:|
| 主课程学习 | 4-5 小时 |
| 项目开发 | 5-6 小时 |
| 算法练习 | 3 小时 |
| 笔记整理 | 2 小时 |
| 博客发布或草稿 | 1.5-2 小时 |
| 通勤轻学习 | 6-8 小时 |

工作日晚上只做 60-90 分钟深度任务。周五主要用于清洁、整理草稿和休息。周末上午做难内容，下午做项目。

## 学习主线

### 阶段 0：工具和开发环境

重点是 Git、Shell、命令行、项目本地启动。

产出：

- 跑通博客源码。
- 写清楚本地开发流程。
- 能稳定提交代码和写开发日志。

### 阶段 1：Web、JS、SQL 基础

重点是 HTML、CSS、JavaScript、Promise、HTTP、SQL。

产出：

- 改善博客文章页结构。
- 检查 API 错误处理。
- 画出博客数据库表关系。
- 写几篇基础复盘文章。

### 阶段 2：算法与数据结构

重点是复杂度、排序、哈希、链表、树、图、动态规划。

产出：

- 建立算法题解分类。
- 每周至少 2 道题。
- 每周整理短题解。

### 阶段 3：全栈工程

重点是 React、TypeScript、后端 API、测试、Docker、部署。

产出：

- 博客系统 README 更完整。
- 增加测试和构建检查。
- 补部署文档。
- 项目逐渐变成可展示作品。

### 阶段 4：系统、网络、数据库深化

重点是操作系统、网络、数据库性能、索引和缓存。

产出：

- 分析从输入 `kpblog.cc` 到页面展示的完整链路。
- 对文章列表、搜索、评论等接口做性能分析。
- 写一次真实问题定位记录。

### 阶段 5：AI 博客助手

最后把 AI 接入博客和笔记系统。

目标功能：

- 基于博客文章问答。
- 基于 Obsidian 笔记问答。
- 自动总结文章。
- 自动推荐标签。
- 根据学习复盘推荐下周重点。

这个阶段要重视权限、来源引用、日志和失败案例，不能只做一个表面 Demo。

## 检验方式

每周至少检查：

```text
[ ] 算法 2 道以上
[ ] 博客系统至少 1 次有效 commit
[ ] Obsidian 笔记 3 次以上
[ ] 博客草稿或公开文章 1 篇
[ ] 周日完成复盘
[ ] 至少 1 个晚上完全休息
```

如果连续两周完成率低于 60%，就减少课程输入，保留项目和算法。如果只看课不写代码，下一周课程减半，项目时间翻倍。

## 总结

这份计划的核心是闭环：

学习一个知识点，马上在博客系统里找应用场景；遇到问题写进 Obsidian；解决后整理成博客；周末复盘，再调整下周计划。

长期坚持下来，我希望这个博客不仅是内容展示站，也能成为我工程能力成长的证据。
